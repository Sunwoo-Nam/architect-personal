# /project:new-diagram

Architecture View 다이어그램을 draw.io XML 형식으로 생성해 `diagrams/` 에 저장한다.

## 사용법
```
/project:new-diagram <다이어그램-타입> [설명]
```

### 다이어그램 타입
- `component` — 컴포넌트 뷰 (박스+화살표, 인터페이스 경계 포함)
- `sequence <시나리오명>` — 시퀀스 다이어그램
- `context` — 시스템 컨텍스트 다이어그램 (C4 Level 1)
- `container` — 컨테이너 다이어그램 (C4 Level 2)
- `dataflow` — 데이터 흐름 다이어그램
- `deployment` — 배포 뷰 (TV 디바이스 + 클라우드)
- `state <컴포넌트명>` — 상태 머신 다이어그램

## 동작 방식

1. `$ARGUMENTS`에서 타입과 설명을 파싱한다.
2. 연관 문서(해당 뷰와 관련된 FR, 컴포넌트 목록)를 읽어 컨텍스트를 파악한다.
3. 타입에 맞는 draw.io XML을 생성한다.
   - 파일명: `diagrams/NN-{purpose}.drawio` (NN은 기존 파일 다음 번호)
   - 스타일: 모노크롬 기반, 컴포넌트 경계는 두꺼운 선, 외부 시스템은 점선 경계
   - 레이어: 각 관심사별로 레이어 분리
4. 파일을 저장하고 관련 문서에 참조 링크를 추가한다.
5. 사용자에게 커밋 여부를 묻는다.

## 주의
- draw.io에서 직접 열 수 있는 유효한 XML이어야 한다.
- 다이어그램 요소는 문서에 이미 정의된 컴포넌트/영역 명칭을 그대로 사용한다.
- 컨텍스트에 맞는 추상화 수준을 유지한다 (너무 세부적이거나 너무 추상적이지 않게).
