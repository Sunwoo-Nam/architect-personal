# Use Case 카탈로그 — Functional Requirements 기반

> [draft.md](draft.md)의 FR을 **행위 흐름(Use Case)** 관점으로 재구성한 문서.
> 각 UC는 사용자/외부 시스템/개발자/운영자/규제자가 목표를 달성하는 흐름을 기술하며,
> **관련 FR**로 추적성을 유지한다.
> 형식: UseCase Name / Precondition / Basic + Alternative Flow / Exceptional Flow.

> **범위 조정 (2026-06-04)**: FR draft 정비에 연동하여 **UC-09(멀티프로필)·UC-13(페이지 인식)·UC-14(조건부 활성화)·UC-22(비자동화 대체 경로)**를 삭제했다(해당 FR 삭제·이관에 따름). UC 번호는 재배정하지 않고 결번으로 둔다. UC-17의 권한 최소화는 NFR-012로 이관, UC-22(부가성 원칙)는 법적 구속력 부재로 완전 삭제.
>
> **범위 조정 (GWP 제외)**: GWP(Generative Web Page) 기능의 과제 범위 제외에 따라 **UC-08(GWP 재구성·표시)·UC-28(GWP 트리거)·UC-29(원본↔GWP 토글)**를 삭제했다(FR-034~038·NFR-031~033 삭제에 따름). UC 번호는 결번으로 둔다.

## A. 사용자 대면 에이전트 흐름

### UC-01 음성 발화 기반 멀티도메인 태스크 수행
**관련 FR**: FR-001, FR-002, FR-003, FR-007

- **Precondition**
  - 에이전트 런타임이 활성 상태이고 외부 음성 인식(STT) 시스템과 연결되어 있다.
  - 사용자 프로필 컨텍스트가 결정되어 있다.
- **Basic Flow**
  1. 시스템이 외부 STT로부터 전사된 발화를 수신한다.
  2. 사용자 의도를 해석하고 태스크를 계획한다.
  3. 검색·조회·예약·구매·콘텐츠 탐색·스마트홈 등 도메인에 걸쳐 단일 세션으로 단계를 실행한다.
  4. 각 단계(페이지 분석·입력·응답 대기)를 실시간으로 표시한다.
  5. 태스크를 완료하고 결과를 사용자에게 제시한다.
- **Alternative Flow**
  - **A1 (의도 모호)**: 의도가 모호하면 사용자에게 재질의하여 명확화 후 2단계로 복귀한다.
- **Exceptional Flow**
  - **E1 (의도 해석 실패)**: 의도를 해석할 수 없으면 수동 입력 안내로 폴백한다. *(FR-015)*
  - **E2 (미지원 도메인)**: 계획 단계에서 지원 불가 도메인이 식별되면 사유를 고지하고 대체 경로를 제시한다. *(FR-015)*

### UC-02 구매·예약 End-to-end 자동 완료
**관련 FR**: FR-003, FR-004, FR-005

- **Precondition**
  - 외부에 변경을 확정하는 태스크(구매·예약 등)가 식별되었고 대상 서비스에 접근 가능하다.
- **Basic Flow**
  1. 사용자가 구매·예약 등 외부 동작을 확정하는 태스크를 요청한다.
  2. 시스템이 페이지 조작 없이 필요한 단계를 자동 수행한다.
  3. 결제·구독 등 **비가역 동작 직전** 확인 단계를 제시한다.
  4. 사용자가 명시적으로 승인한다.
  5. 시스템이 완료까지 실행하고 결과를 반환한다.
- **Alternative Flow**
  - **A1 (반복 결제·구독)**: 구독 등록 전 반복 조건(금액·주기·다음 결제일)을 명시하고, 등록 후 해지 수단으로 안내한다. *(FR-005 AC)*
- **Exceptional Flow**
  - **E1 (사용자 거부)**: 확인 단계에서 거부하면 동작을 중단하고 안전 상태로 복귀한다.
  - **E2 (실행 실패)**: 단계 실패 시 원본 페이지 이동 또는 수동 안내로 폴백한다. *(FR-015)*

### UC-03 로그인·인증 장벽 통과
**관련 FR**: FR-012

- **Precondition**: 에이전트가 태스크 실행 중이며 자격증명이 플랫폼/브라우저 자격증명 저장소에 보관되어 있다(저장은 본 시스템 범위 외).
- **Basic Flow**
  1. 태스크 실행 중 로그인 페이지/캡차를 감지한다.
  2. 플랫폼/브라우저가 보관한 자격증명 또는 플랫폼 제공 인증으로 자동 처리를 시도한다.
  3. 인증을 통과하고 태스크를 계속한다.
- **Alternative Flow**
  - **A1 (자동 처리 불가)**: 자동 처리가 불가능하면 실행을 중단하고 Human-in-the-Loop 단계로 사용자에게 인증을 넘긴다. 사용자 인증 완료 후 태스크를 재개한다.
- **Exceptional Flow**
  - **E1 (자격증명 만료/거부)**: 인증 실패가 반복되면 태스크를 중단하고 사용자에게 사유를 알린다.

### UC-04 실행 중 개입 — 취소·재시도·제어권 전환
**관련 FR**: FR-006, FR-008

- **Precondition**: 에이전트가 태스크를 실행 중이다.
- **Basic Flow**
  1. 사용자가 실행 중 현재 동작 취소를 요청한다.
  2. 시스템이 현재 동작을 안전하게 중단한다.
- **Alternative Flow**
  - **A1 (수정 재시도)**: 사용자가 수정된 입력으로 재시도를 선택하면 해당 입력으로 태스크를 다시 계획·실행한다.
  - **A2 (Co-navigation)**: 사용자가 직접 조작을 원하면 세션을 종료하지 않고 페이지 조작 제어권을 인수하며, 이후 다시 에이전트에 위임할 수 있다. *(FR-008)*
- **Exceptional Flow**
  - **E1 (비가역 직전 취소 불가)**: 비가역 동작 커밋 시점에는 취소가 불가하며, 시스템은 확인 단계로 회귀시킨다. *(FR-005)*

---

### UC-05 백그라운드 실행 및 멀티태스킹
**관련 FR**: FR-009, FR-007

- **Precondition**: 장시간 소요 태스크가 실행 중이다.
- **Basic Flow**
  1. 사용자가 다른 TV 기능(VOD 시청·채널 변경)으로 전환한다.
  2. 시스템이 에이전트 태스크를 백그라운드에서 중단 없이 계속한다.
  3. 완료 시 사용자에게 결과/알림을 표시한다.
- **Alternative Flow**
  - **A1 (진행 확인)**: 사용자가 진행 상황 조회를 요청하면 현재 단계를 실시간 표시한다. *(FR-007)*
- **Exceptional Flow**
  - **E1 (백그라운드 중 실패)**: 실패 시 폴백 경로와 함께 사용자에게 알린다. *(FR-015)*

---

### UC-06 Headless 실행 및 Headed 전환
**관련 FR**: FR-011, FR-013

- **Precondition**: 가시적 브라우저 화면이 불필요한 태스크(정보 조회·예약 확인 등)이다.
- **Basic Flow**
  1. 시스템이 브라우저 페이지를 렌더링하지 않고 태스크를 처리한다.
  2. 결과만 반환한다.
- **Alternative Flow**
  - **A1 (Headed 전환)**: 처리 중/완료 후 사용자가 페이지 보기를 요청하면 태스크를 재시작하지 않고 전체 페이지를 Headed 모드로 렌더링한다. *(FR-013)*
- **Exceptional Flow**
  - **E1 (렌더링 불가)**: Headed 전환에 실패하면 결과 요약만 유지하고 사유를 고지한다.

---

### UC-07 네트워크 단절 후 태스크 재개
**관련 FR**: FR-010

- **Precondition**: 에이전트 태스크 실행 중이다.
- **Basic Flow**
  1. 태스크 실행 중 네트워크 연결이 끊긴다.
  2. 시스템이 현재 태스크 상태를 보존한다.
  3. 재연결 시 마지막 안정 체크포인트부터 재개한다.
- **Alternative Flow**
  - **A1 (장기 단절)**: 일정 시간 내 재연결되지 않으면 사용자에게 보류 상태를 통지하고 수동 재개 옵션을 제공한다.
- **Exceptional Flow**
  - **E1 (상태 복원 실패)**: 체크포인트 복원이 불가하면 태스크를 안전 종료하고 처음부터 재시도를 안내한다. *(FR-015)*

### UC-10 에이전트 행동 설명 요청 (Explainability)
**관련 FR**: FR-033

- **Precondition**: 에이전트가 동작을 수행했거나 수행 중이다.
- **Basic Flow**
  1. 사용자가 특정 에이전트 행동의 이유 설명을 요청한다.
  2. 시스템이 간결하고 이해 가능한 근거를 제시한다.
- **Alternative Flow**
  - **A1 (행동 추적)**: 사용자가 추가 단계 설명을 요청하면 직전 단계의 의사결정 맥락을 이어서 제시한다.
- **Exceptional Flow**
  - **E1 (근거 부재)**: 설명 가능한 근거 정보가 없으면 그 사실을 고지한다.

### UC-11 개인화 — 히스토리·선호 누적 및 활용
**관련 FR**: FR-016

- **Precondition**: 사용자 프로필이 활성이고 이력 누적이 허용된 상태이다.
- **Basic Flow**
  1. 시스템이 브라우저 이력·과거 에이전트 요청·태스크 결과를 누적한다.
  2. 이후 태스크 계획 시 누적 컨텍스트를 적용해 정확도를 높이고 필요한 사용자 입력을 줄인다.
- **Alternative Flow**
  - **A1 (컨텍스트 미흡)**: 누적 데이터가 부족하면 기본 동작으로 처리하고 점진적으로 컨텍스트를 축적한다.
- **Exceptional Flow**
  - **E1 (이력 비활성)**: 사용자가 이력 누적을 비활성화한 경우 개인화 없이 처리한다.

### UC-12 접근성 표시 옵션 적용
**관련 FR**: FR-017

- **Precondition**: 접근성 표시 설정(텍스트 크기·대비·음성 응답 속도)이 구성되어 있다.
- **Basic Flow**
  1. 사용자가 접근성 표시 옵션을 설정한다.
  2. 시스템이 모든 출력(시각·음성)에 해당 설정을 적용한다.
- **Alternative Flow**
  - **A1 (실시간 변경)**: 세션 중 설정을 변경하면 즉시 후속 출력에 반영한다.
- **Exceptional Flow**
  - **E1 (미지원 옵션)**: 특정 출력이 옵션을 지원하지 못하면 가장 근접한 표현으로 대체하고 한계를 고지한다.

### UC-15 전원 상태 변화에 따른 에이전트 생명주기 관리
**관련 FR**: FR-023

- **Precondition**: 에이전트 하네스가 동작 중이고 TV 플랫폼 생명주기 이벤트를 수신한다.
- **Basic Flow**
  1. TV가 슬립 모드 진입 이벤트를 발생시킨다.
  2. 시스템이 에이전트 하네스를 안전하게 일시 중단한다.
  3. 슬립 복귀 시 하네스를 복원한다.
- **Alternative Flow**
  - **A1 (진행 중 태스크)**: 중단 시 진행 중 태스크는 체크포인트로 보존하고 복귀 후 재개를 시도한다. *(FR-010)*
- **Exceptional Flow**
  - **E1 (복원 실패)**: 복원에 실패하면 하네스를 안전 종료하고 다음 세션에서 재초기화한다.

---

### UC-16 에이전트 오류 격리 및 상위 시스템 보호
**관련 FR**: FR-024

- **Precondition**: 에이전트 하네스가 상위 Main Agent / Generative UI Renderer와 통합된 상태로 동작 중이다.
- **Basic Flow**
  1. 에이전트 하네스에서 복구 불가능한 오류가 발생한다.
  2. 시스템이 오류를 격리한다.
  3. 상위 Main Agent 또는 Generative UI Renderer로 전파되지 않도록 차단한다.
- **Alternative Flow**
  - **A1 (부분 복구)**: 격리 후 하네스 일부를 재시작해 세션 연속성을 유지할 수 있으면 복구한다.
- **Exceptional Flow**
  - **E1 (격리 실패)**: 격리에 실패하면 하네스 전체를 종료하고 상위 시스템에 안전 실패 신호를 보낸다.

## C. 보안·프라이버시

### UC-17 자격증명 비노출 사용 및 권한 최소화
**관련 NFR**: NFR-012 (권한 최소화), NFR-018 (자격증명 비전송)

- **Precondition**: 태스크 실행을 위해 (플랫폼/브라우저가 보관한) 자격증명 또는 시스템 권한이 필요하다. *(자격증명 저장 자체는 본 시스템 범위 외)*
- **Basic Flow**
  1. 현재 태스크에 필요한 최소 권한만 요청·행사한다. *(NFR-012)*
  2. 더 이상 필요하지 않은 권한을 해제한다.
- **Alternative Flow**
  - **A1 (자격증명 사용)**: 자격증명이 필요하면 AI 모델 컨텍스트에 노출하지 않고 의도된 범위 내에서만 사용한다. *(NFR-018)*
- **Exceptional Flow**
  - **E1 (범위 외 전송 시도)**: 의도된 범위를 벗어난 자격증명 전송 시도를 차단하고 기록한다.

---

### UC-18 프롬프트 인젝션 탐지 및 방어
**관련 FR**: FR-020

- **Precondition**: 에이전트가 외부 웹 콘텐츠를 컨텍스트로 처리 중이다.
- **Basic Flow**
  1. 시스템이 외부 콘텐츠에서 에이전트 동작 탈취·유도 명령을 탐지한다.
  2. 해당 명령을 무시(폐기)한다.
  3. 사용자에게 탐지 사실을 알린다.
- **Alternative Flow**
  - **A1 (의심 콘텐츠)**: 탈취 여부가 모호하면 보수적으로 격리하고 해당 콘텐츠 기반 동작을 보류한다.
- **Exceptional Flow**
  - **E1 (방어 우회 정황)**: 방어 우회 정황이 감지되면 태스크를 중단하고 안전 상태로 복귀한다. *(FR-024)*

---

### UC-19 PII 분류·보존 기간 제어 및 자동 삭제
**관련 FR**: FR-019

- **Precondition**: 시스템이 음성 전사·DOM 스냅샷·화면 콘텐츠 등 데이터를 생성/수집한다.
- **Basic Flow**
  1. 시스템이 잠재적 PII 포함 데이터를 분류한다.
  2. 설정된 보존 기간을 적용한다.
  3. 보존 기간 만료 시 데이터를 자동 삭제한다.
- **Alternative Flow**
  - **A1 (즉시 삭제 요청)**: 사용자가 삭제 권리를 행사하면 보존 기간과 무관하게 즉시 삭제한다. *(FR-021)*
- **Exceptional Flow**
  - **E1 (삭제 실패)**: 자동 삭제에 실패하면 재시도 큐에 등록하고 운영자에게 알린다.

---

## D. 규제·컴플라이언스

### UC-20 개인정보 동의 수집 및 데이터 주체 권리 행사
**관련 FR**: FR-021

- **Precondition**: 시스템이 개인정보 수집을 시작하려 한다.
- **Basic Flow**
  1. 개인정보 수집 전에 사용자 동의를 요청한다.
  2. 사용자가 동의하면 수집을 진행한다.
  3. 사용자가 언제든지 데이터 주체 권리(열람·삭제·이전)를 행사할 수 있는 수단을 제공한다.
- **Alternative Flow**
  - **A1 (권리 행사)**: 사용자가 열람/삭제/이전을 요청하면 해당 요청을 처리한다. *(FR-021)*
- **Exceptional Flow**
  - **E1 (동의 거부)**: 동의를 거부하면 해당 데이터를 수집하지 않고 가능한 범위의 기능만 제공한다.

---

### UC-21 데이터 처리 방식 사용자 고지 (온디바이스 vs 클라우드)
**관련 FR**: FR-022

- **Precondition**: 태스크 처리를 위해 사용자 데이터를 온디바이스 외부 클라우드로 전송해야 한다.
- **Basic Flow**
  1. 시스템이 클라우드 전송 전에 데이터가 디바이스 외부에서 처리됨을 고지한다.
  2. 사용자가 확인(acknowledgment)한다.
  3. 시스템이 데이터를 전송하고 태스크를 처리한다.
- **Alternative Flow**
  - **A1 (온디바이스 처리 가능)**: 온디바이스로 처리 가능한 경우 고지 없이 디바이스 내에서 처리한다.
- **Exceptional Flow**
  - **E1 (확인 거부)**: 사용자가 확인을 거부하면 클라우드 처리를 중단하고 대체(온디바이스/수동) 경로를 안내한다.

## E. 외부 통합 (플랫폼·제3자)

### UC-23 외부 에이전트 플랫폼의 Skill 호출
**관련 FR**: FR-025, FR-026, FR-027

- **Precondition**: 외부 에이전트 플랫폼이 표준 인터페이스로 본 시스템을 등록했다.
- **Basic Flow**
  1. 외부 플랫폼이 표준 인터페이스로 에이전트를 호출한다.
  2. 시스템이 호출자를 인증하고 권한 범위를 검증한다. *(FR-026)*
  3. 태스크를 실행한다.
  4. 실행 상태(성공·실패·부분)·결과·에러 코드를 구조화 스키마로 반환한다. *(FR-027)*
- **Alternative Flow**
  - **A1 (부분 완료)**: 일부만 완료되면 partial 상태와 함께 완료/미완료 단계를 반환한다.
- **Exceptional Flow**
  - **E1 (인증·권한 실패)**: 인증 실패 또는 권한 범위 위반 시 실행 전 거부하고 에러 코드를 반환한다. *(FR-026)*

---

### UC-24 공개 인터페이스 버전 관리 및 하위 호환성
**관련 FR**: FR-025 / **관련 NFR**: NFR-023 (하위 호환성 유지 기간)

- **Precondition**: 외부 호출자가 특정 버전의 공개 인터페이스에 의존한다.
- **Basic Flow**
  1. 시스템이 모든 공개 인터페이스에 버전 식별자를 부여한다.
  2. 호출자가 지정 버전으로 호출하면 해당 버전 계약대로 응답한다.
  3. 최소 이전 메이저 버전과의 하위 호환성을 유지한다.
- **Alternative Flow**
  - **A1 (구버전 호출)**: 이전 메이저 버전 호출은 호환 계층을 통해 처리한다.
- **Exceptional Flow**
  - **E1 (지원 종료 버전)**: 호환 범위를 벗어난 버전 호출은 거부하고 마이그레이션 정보를 반환한다.

---

### UC-25 제3자 웹 서비스 접근 — 신원 표명 및 정책 준수
**관련 FR**: FR-028, FR-029

- **Precondition**: 에이전트가 제3자 웹 서비스에 자동화를 수행하려 한다.
- **Basic Flow**
  1. 자동화 전에 서비스의 에이전트 접근 정책(robots.txt·접근 제어 헤더)을 확인한다. *(FR-029)*
  2. 일반 브라우저와 구별 가능한 Agent-ID/커스텀 User-Agent로 자신을 식별한다. *(FR-028)*
  3. 선언된 rate limit·crawl delay를 준수하며 태스크를 수행한다.
- **Alternative Flow**
  - **A1 (제한 정책)**: 정책이 일부 동작을 제한하면 허용 범위 내에서만 수행하고 나머지는 사용자에게 위임한다. *(FR-012)*
- **Exceptional Flow**
  - **E1 (접근 거부 정책)**: 정책이 에이전트 접근을 금지하면 자동화를 중단하고 사용자에게 수동 경로를 안내한다.

## F. 개발·운영·테스트

### UC-26 개발자 디버깅 — 실행 트레이스 및 계측 조회
**관련 FR**: FR-030, FR-031

- **Precondition**: 개발자/운영자가 에이전트 실행 데이터에 접근 권한을 가진다.
- **Basic Flow**
  1. 시스템이 에이전트 루프 각 이터레이션의 구조화 트레이스(도구 호출·컨텍스트 스냅샷·모델 응답·에러 지점)를 기록한다. *(FR-030)*
  2. 요청별 토큰 사용량·모델 응답 지연·예상 비용을 계측한다. *(FR-031)*
  3. 개발자/운영자가 접근 가능한 형식으로 트레이스·메트릭을 노출한다.
- **Alternative Flow**
  - **A1 (실시간 모니터링)**: 운영자가 실시간 조회를 요청하면 스트리밍 형태로 메트릭을 제공한다.
- **Exceptional Flow**
  - **E1 (계측 누락)**: 일부 메트릭 수집에 실패하면 누락을 표시하고 부분 데이터를 노출한다.

---

### UC-27 시뮬레이션·Mock 기반 격리 테스트
**관련 FR**: FR-032 / **관련 NFR**: NFR-026 (Mock 적용 범위)

- **Precondition**: QA/개발자가 실제 외부 서비스·사용자 데이터에 영향을 주지 않고 테스트하려 한다.
- **Basic Flow**
  1. 시뮬레이션 모드를 구성한다.
  2. 시스템이 샌드박스/목(mock) 브라우저 환경에서 에이전트 태스크를 실행한다. *(FR-032)*
  3. 각 기능 경계(VUI·에이전트 코어·브라우저 제어·모델 추상화)의 Mock/Stub로 격리 단위·통합 테스트를 수행한다. *(NFR-026)*
- **Alternative Flow**
  - **A1 (부분 Mock)**: 일부 경계만 Mock으로 대체하고 나머지는 실제 구현으로 통합 테스트한다.
- **Exceptional Flow**
  - **E1 (실제 서비스 접촉)**: 시뮬레이션 모드에서 실제 외부 서비스 접촉이 감지되면 호출을 차단하고 테스트를 실패 처리한다.
