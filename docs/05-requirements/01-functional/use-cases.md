# Use Case 카탈로그 — Functional Requirements 기반

> [draft.md](draft.md)의 FR을 **행위 흐름(Use Case)** 관점으로 재구성한 문서.
> 각 UC는 사용자/외부 시스템/개발자/운영자/규제자가 목표를 달성하는 흐름을 기술하며,
> **관련 FR**로 추적성을 유지한다.
> 형식: UseCase Name / Actor / Precondition / Basic + Alternative Flow / Exceptional Flow.
> **Actor 표기**: 주(primary) actor를 먼저, 보조(secondary) actor·트리거를 괄호로 표기한다.

> **FR-015 (폴백) 표현 방식**: 에이전트 실패 폴백은 특정 UC가 소유하지 않는 횡단 행위로, 개별 UC의 Exceptional Flow에 분산 표현한다(UC-01·02·05·06 E-flow 참조). 별도 UC를 두지 않는 것은 의도된 결정이다.

![Use Case Diagram](../../../diagrams/03-use-case-diagram.drawio)

> 위 다이어그램은 본 카탈로그의 활성 UC 17개를 A~F 그룹·Actor 연관과 함께 시각화한 것이다. UC 추가·삭제·Actor 변경 시 다이어그램을 함께 갱신한다.

> **개정 이력 (2026-07-11 UC 축소·재번호)**: 다이어그램 정비에서 UC 6개를 제외하고, 남은 활성 UC를 문서 순서대로 **UC-01~UC-17**로 재배정했다. 하위 문서([03-test-cases.md](../03-test-cases.md)의 TC-UC-NN)와 [다이어그램](../../../diagrams/03-use-case-diagram.drawio)을 함께 갱신했다.
> - **제외된 UC(직전 번호)**: UC-06 Headless 실행 및 Headed 전환, UC-08 에이전트 행동 설명(Explainability), UC-10 접근성 표시 옵션, UC-12 전원 상태 생명주기 관리, UC-18 데이터 처리 방식 고지, UC-21 제3자 웹 서비스 접근 정책 준수. 제외된 UC가 참조하던 FR은 카탈로그 차원에서만 UC 흐름을 제거한 것이며, FR 자체의 존치·삭제는 [draft.md](draft.md)를 단일 진실원으로 따른다.
> - **재번호 매핑 (직전→현행)**: 01–05 유지 · 07→06 · 09→07 · 11→08 · 13→09 · 14→10 · 15→11 · 16→12 · 17→13 · 19→14 · 20→15 · 22→16 · 23→17.

## A. 사용자 대면 에이전트 흐름

### UC-01 음성 발화 기반 멀티도메인 태스크 수행
**관련 FR**: FR-001, FR-002, FR-003, FR-007

- **Actor**: 사용자 (외부 음성 인식 시스템 Bixby→argot — 발화 전사 전달, 제3자 웹 서비스 — 태스크 대상)
- **Precondition**
  - 에이전트 런타임이 활성 상태이고 외부 음성 인식(STT) 시스템과 연결되어 있다.
  - 사용자 프로필 컨텍스트가 결정되어 있다.
- **Basic Flow**
  1. 시스템이 외부 STT로부터 전사된 발화를 수신한다.
  2. 사용자 의도를 해석하고 태스크를 계획한다.
  3. 검색·조회·예약·구매·콘텐츠 탐색·스마트홈 등 도메인에 걸쳐 단일 세션으로 단계를 실행한다.
  4. 각 단계(페이지 분석·입력·응답 대기)를 실시간으로 표시한다.
  5. 태스크를 완료하고 결과를 사용자에게 제시한다.
- **Alternative Flow**
  - **A1 (의도 모호)**: 의도가 모호하면 사용자에게 재질의하여 명확화 후 2단계로 복귀한다.
- **Exceptional Flow**
  - **E1 (의도 해석 실패)**: 의도를 해석할 수 없으면 수동 입력 안내로 폴백한다. *(FR-015)*
  - **E2 (미지원 도메인)**: 계획 단계에서 지원 불가 도메인이 식별되면 사유를 고지하고 대체 경로를 제시한다. *(FR-015)*

### UC-02 구매·예약 End-to-end 자동 완료
**관련 FR**: FR-003, FR-004, FR-005

- **Actor**: 사용자 (제3자 웹 서비스 — 구매·예약 대상)
- **Precondition**
  - 외부에 변경을 확정하는 태스크(구매·예약 등)가 식별되었고 대상 서비스에 접근 가능하다.
- **Basic Flow**
  1. 사용자가 구매·예약 등 외부 동작을 확정하는 태스크를 요청한다.
  2. 시스템이 페이지 조작 없이 필요한 단계를 자동 수행한다.
  3. 결제·구독 등 **비가역 동작 직전** 확인 단계를 제시한다.
  4. 사용자가 명시적으로 승인한다.
  5. 시스템이 완료까지 실행하고 결과를 반환한다.
- **Alternative Flow**
  - **A1 (반복 결제·구독)**: 구독 등록 전 반복 조건(금액·주기·다음 결제일)을 명시하고, 등록 후 해지 수단으로 안내한다. *(FR-005 AC)*
- **Exceptional Flow**
  - **E1 (사용자 거부)**: 확인 단계에서 거부하면 동작을 중단하고 안전 상태로 복귀한다.
  - **E2 (실행 실패)**: 단계 실패 시 원본 페이지 이동 또는 수동 안내로 폴백한다. *(FR-015)*

### UC-03 로그인·인증 장벽 통과
**관련 FR**: FR-012

- **Actor**: 사용자 — HITL 인증 인계 시 (제3자 웹 서비스 — 인증 장벽 제시, 트리거)
- **Precondition**: 에이전트가 태스크 실행 중이며 자격증명이 플랫폼/브라우저 자격증명 저장소에 보관되어 있다(저장은 본 시스템 범위 외).
- **Basic Flow**
  1. 태스크 실행 중 로그인 페이지/캡차를 감지한다.
  2. 플랫폼/브라우저가 보관한 자격증명 또는 플랫폼 제공 인증으로 자동 처리를 시도한다.
  3. 인증을 통과하고 태스크를 계속한다.
- **Alternative Flow**
  - **A1 (자동 처리 불가)**: 자동 처리가 불가능하면 실행을 중단하고 Human-in-the-Loop 단계로 사용자에게 인증을 넘긴다. 사용자 인증 완료 후 태스크를 재개한다.
- **Exceptional Flow**
  - **E1 (자격증명 만료/거부)**: 인증 실패가 반복되면 태스크를 중단하고 사용자에게 사유를 알린다.

### UC-04 실행 중 개입 — 취소·재시도·제어권 전환
**관련 FR**: FR-006, FR-008

- **Actor**: 사용자
- **Precondition**: 에이전트가 태스크를 실행 중이다.
- **Basic Flow**
  1. 사용자가 실행 중 현재 동작 취소를 요청한다.
  2. 시스템이 현재 동작을 안전하게 중단한다.
- **Alternative Flow**
  - **A1 (수정 재시도)**: 사용자가 수정된 입력으로 재시도를 선택하면 해당 입력으로 태스크를 다시 계획·실행한다.
  - **A2 (Co-navigation)**: 사용자가 직접 조작을 원하면 세션을 종료하지 않고 페이지 조작 제어권을 인수하며, 이후 다시 에이전트에 위임할 수 있다. *(FR-008)*
- **Exceptional Flow**
  - **E1 (비가역 직전 취소 불가)**: 비가역 동작 커밋 시점에는 취소가 불가하며, 시스템은 확인 단계로 회귀시킨다. *(FR-005)*

### UC-05 백그라운드 실행 및 멀티태스킹
**관련 FR**: FR-009, FR-007

- **Actor**: 사용자
- **Precondition**: 장시간 소요 태스크가 실행 중이다.
- **Basic Flow**
  1. 사용자가 다른 TV 기능(VOD 시청·채널 변경)으로 전환한다.
  2. 시스템이 에이전트 태스크를 백그라운드에서 중단 없이 계속한다.
  3. 완료 시 사용자에게 결과/알림을 표시한다.
- **Alternative Flow**
  - **A1 (진행 확인)**: 사용자가 진행 상황 조회를 요청하면 현재 단계를 실시간 표시한다. *(FR-007)*
- **Exceptional Flow**
  - **E1 (백그라운드 중 실패)**: 실패 시 폴백 경로와 함께 사용자에게 알린다. *(FR-015)*

### UC-06 네트워크 단절 후 태스크 재개
**관련 FR**: FR-010

- **Actor**: 시스템 자체 — 네트워크 단절·재연결 이벤트, 트리거 (사용자 — 장기 단절 시 수동 재개)
- **Precondition**: 에이전트 태스크 실행 중이다.
- **Basic Flow**
  1. 태스크 실행 중 네트워크 연결이 끊긴다.
  2. 시스템이 현재 태스크 상태를 보존한다.
  3. 재연결 시 마지막 안정 체크포인트부터 재개한다.
- **Alternative Flow**
  - **A1 (장기 단절)**: 일정 시간 내 재연결되지 않으면 사용자에게 보류 상태를 통지하고 수동 재개 옵션을 제공한다.
- **Exceptional Flow**
  - **E1 (상태 복원 실패)**: 체크포인트 복원이 불가하면 태스크를 안전 종료하고 처음부터 재시도를 안내한다. *(FR-015)*

### UC-07 개인화 — 히스토리·선호 누적 및 활용
**관련 FR**: FR-016

- **Actor**: 시스템 자체 — 이력 누적·적용 (사용자 — 누적 허용·비활성 설정)
- **Precondition**: 사용자 프로필이 활성이고 이력 누적이 허용된 상태이다.
- **Basic Flow**
  1. 시스템이 브라우저 이력·과거 에이전트 요청·태스크 결과를 누적한다.
  2. 이후 태스크 계획 시 누적 컨텍스트를 적용해 정확도를 높이고 필요한 사용자 입력을 줄인다.
- **Alternative Flow**
  - **A1 (컨텍스트 미흡)**: 누적 데이터가 부족하면 기본 동작으로 처리하고 점진적으로 컨텍스트를 축적한다.
- **Exceptional Flow**
  - **E1 (이력 비활성)**: 사용자가 이력 누적을 비활성화한 경우 개인화 없이 처리한다.

### UC-08 Quick Action — 에이전트 루프 우회 빠른 경로 실행
**관련 FR**: FR-003a / **관련 NFR**: NFR-001 (단순 명령 1초 이내 응답)

- **Actor**: 사용자 (외부 음성 인식 시스템 Bixby→argot — 발화 전사 전달)
- **Precondition**: 에이전트 런타임이 활성 상태이고 Quick Action 카탈로그가 로드되어 있다.
- **Basic Flow**
  1. 사용자가 사전 정의된 Quick Action에 해당하는 명령(앱 실행, 채널 변경, 날씨 조회, 직접 디바이스 제어 등)을 발화한다.
  2. 시스템이 입력을 Quick Action 카탈로그와 대조하여 일치를 판정한다.
  3. LLM 기반 에이전트 추론 루프를 호출하지 않고 전용 빠른 경로로 결정론적으로 실행한다. *(FR-003a)*
  4. 결과를 즉시 표시하거나 동작을 완료한다. *(NFR-001)*
- **Alternative Flow**
  - **A1 (분류 모호)**: Quick Action 분류 결과가 모호하면 일반 에이전트 경로(UC-01)로 폴백한다. *(FR-003a AC 2)*
- **Exceptional Flow**
  - **E1 (실행 실패)**: Quick Action 실행이 실패하면 폴백 경로를 제공한다. *(FR-015, FR-003a AC 3)*

## B. 플랫폼 통합·생명주기 (시스템 내부)

### UC-09 에이전트 오류 격리 및 상위 시스템 보호
**관련 FR**: FR-024

- **Actor**: 시스템 자체 — 복구 불가 오류 이벤트, 트리거 (argot Main Agent / GenUI Renderer — 보호 대상 상위 호스트)
- **Precondition**: 에이전트 하네스가 상위 Main Agent / Generative UI Renderer와 통합된 상태로 동작 중이다.
- **Basic Flow**
  1. 에이전트 하네스에서 복구 불가능한 오류가 발생한다.
  2. 시스템이 오류를 격리한다.
  3. 상위 Main Agent 또는 Generative UI Renderer로 전파되지 않도록 차단한다.
- **Alternative Flow**
  - **A1 (부분 복구)**: 격리 후 하네스 일부를 재시작해 세션 연속성을 유지할 수 있으면 복구한다.
- **Exceptional Flow**
  - **E1 (격리 실패)**: 격리에 실패하면 하네스 전체를 종료하고 상위 시스템에 안전 실패 신호를 보낸다.

## C. 보안·프라이버시

### UC-10 자격증명 비노출 사용 및 권한 최소화
**관련 NFR**: NFR-012 (권한 최소화), NFR-018 (자격증명 비전송)

- **Actor**: 시스템 자체 — 내부 보안 정책 집행 (보안 · 컴플라이언스팀 — 정책 소유, 3.2.8)
- **Precondition**: 태스크 실행을 위해 (플랫폼/브라우저가 보관한) 자격증명 또는 시스템 권한이 필요하다. *(자격증명 저장 자체는 본 시스템 범위 외)*
- **Basic Flow**
  1. 현재 태스크에 필요한 최소 권한만 요청·행사한다. *(NFR-012)*
  2. 더 이상 필요하지 않은 권한을 해제한다.
- **Alternative Flow**
  - **A1 (자격증명 사용)**: 자격증명이 필요하면 AI 모델 컨텍스트에 노출하지 않고 의도된 범위 내에서만 사용한다. *(NFR-018)*
- **Exceptional Flow**
  - **E1 (범위 외 전송 시도)**: 의도된 범위를 벗어난 자격증명 전송 시도를 차단하고 기록한다.

### UC-11 프롬프트 인젝션 탐지 및 방어
**관련 FR**: FR-020

- **Actor**: 악성 웹 콘텐츠 — 적대적 트리거 (사용자 — 탐지 통지 수신)
- **Precondition**: 에이전트가 외부 웹 콘텐츠를 컨텍스트로 처리 중이다.
- **Basic Flow**
  1. 시스템이 외부 콘텐츠에서 에이전트 동작 탈취·유도 명령을 탐지한다.
  2. 해당 명령을 무시(폐기)한다.
  3. 사용자에게 탐지 사실을 알린다.
- **Alternative Flow**
  - **A1 (의심 콘텐츠)**: 탈취 여부가 모호하면 보수적으로 격리하고 해당 콘텐츠 기반 동작을 보류한다.
- **Exceptional Flow**
  - **E1 (방어 우회 정황)**: 방어 우회 정황이 감지되면 태스크를 중단하고 안전 상태로 복귀한다. *(FR-024)*

### UC-12 PII 분류·보존 기간 제어 및 자동 삭제
**관련 FR**: FR-019

- **Actor**: 시스템 자체 — 보존 기간 만료 시간 트리거 (사용자 — 즉시 삭제 요청, 운영자 — 삭제 실패 알림 수신)
- **Precondition**: 시스템이 음성 전사·DOM 스냅샷·화면 콘텐츠 등 데이터를 생성/수집한다.
- **Basic Flow**
  1. 시스템이 잠재적 PII 포함 데이터를 분류한다.
  2. 설정된 보존 기간을 적용한다.
  3. 보존 기간 만료 시 데이터를 자동 삭제한다.
- **Alternative Flow**
  - **A1 (즉시 삭제 요청)**: 사용자가 삭제 권리를 행사하면 보존 기간과 무관하게 즉시 삭제한다. *(FR-021, UC-13)*
- **Exceptional Flow**
  - **E1 (삭제 실패)**: 자동 삭제에 실패하면 재시도 큐에 등록하고 운영자에게 알린다.

### UC-13 개인정보 동의 수집 및 데이터 주체 권리 행사
**관련 FR**: FR-021

- **Actor**: 사용자 (규제·법률 기관 — 준수 증거의 최종 수신자, 3.2.8이 대변)
- **Precondition**: 시스템이 개인정보 수집을 시작하려 한다.
- **Basic Flow**
  1. 개인정보 수집 전에 사용자 동의를 요청한다.
  2. 사용자가 동의하면 수집을 진행한다.
  3. 사용자가 언제든지 데이터 주체 권리(열람·삭제·이전)를 행사할 수 있는 수단을 제공한다.
- **Alternative Flow**
  - **A1 (권리 행사)**: 사용자가 열람/삭제/이전을 요청하면 해당 요청을 처리한다. *(FR-021; 삭제 실행은 UC-12)*
- **Exceptional Flow**
  - **E1 (동의 거부)**: 동의를 거부하면 해당 데이터를 수집하지 않고 가능한 범위의 기능만 제공한다.

## E. 외부 통합 (플랫폼·제3자)

### UC-14 argot Main Agent의 Sub-Agent / Skill 호출
**관련 FR**: FR-025, FR-026, FR-027 / **관련 제약**: TC-005 (argot 호출 계약), TC-006 (표준 인터페이스)

- **Actor**: argot Main Agent — 주 호출자 (외부 에이전트 플랫폼 — 동일 표준 인터페이스의 잠재 호출자, 3.6 제약)
- **Precondition**: 본 시스템이 argot에 Sub-Agent / Skill로 등록되어 있다. argot이 Bixby를 통해 사용자 질의를 수신했다.
- **Basic Flow**
  1. argot이 사용자 질의를 브라우저 제어 태스크로 판단하고, 표준 인터페이스(버전 관리된 호출 계약)로 본 시스템을 호출한다. *(FR-025)*
  2. 시스템이 호출자를 인증하고 요청이 허가된 권한 범위 내인지 검증한다. *(FR-026)*
  3. 태스크를 실행한다.
  4. 실행 상태(성공·실패·부분)·결과·에러 코드를 구조화 스키마로 argot에 반환하고, argot이 결과를 사용자에게 표시한다. *(FR-027)*
- **Alternative Flow**
  - **A1 (부분 완료)**: 일부만 완료되면 partial 상태와 함께 완료/미완료 단계를 반환한다.
  - **A2 (외부 에이전트 플랫폼 호출)**: MCP 클라이언트 등 외부 에이전트 오케스트레이션 플랫폼이 동일 표준 인터페이스로 호출하는 경우에도 2~4단계가 동일하게 적용된다. *(TC-006; 외부 플랫폼은 Stakeholder가 아닌 제약으로 취급 — 3.6)*
- **Exceptional Flow**
  - **E1 (인증·권한 실패)**: 인증 실패 또는 권한 범위 위반 시 실행 전 거부하고 에러 코드를 반환한다. *(FR-026)*
  - **E2 (하네스 오류)**: 실행 중 복구 불가 오류가 발생하면 오류를 격리하고 argot에 안전 실패 신호를 반환한다. *(FR-024, UC-09)*

### UC-15 공개 인터페이스 버전 관리 및 하위 호환성
**관련 FR**: FR-025 / **관련 NFR**: NFR-023 (하위 호환성 유지 기간)

- **Actor**: argot Main Agent — 주 소비자 (외부 에이전트 플랫폼 — 잠재 소비자, 3.6 제약)
- **Precondition**: 호출자(argot 또는 외부 플랫폼)가 특정 버전의 공개 인터페이스에 의존한다.
- **Basic Flow**
  1. 시스템이 모든 공개 인터페이스에 버전 식별자를 부여한다.
  2. 호출자가 지정 버전으로 호출하면 해당 버전 계약대로 응답한다.
  3. 최소 이전 메이저 버전과의 하위 호환성을 유지하여, argot 및 외부 통합 코드의 마이그레이션 시간을 보장한다. *(NFR-023)*
- **Alternative Flow**
  - **A1 (구버전 호출)**: 이전 메이저 버전 호출은 호환 계층을 통해 처리한다.
- **Exceptional Flow**
  - **E1 (지원 종료 버전)**: 호환 범위를 벗어난 버전 호출은 거부하고 마이그레이션 정보를 반환한다.

### UC-17 시뮬레이션·Mock 기반 격리 테스트
**관련 FR**: FR-032 / **관련 NFR**: NFR-026 (Mock 적용 범위)

- **Actor**: 품질 담당자 (3.2.6) · SW 개발팀 (3.2.4)

- **Basic Flow**

- **Alternative Flow**
  - **A1 (부분 Mock)**: 일부 경계만 Mock으로 대체하고 나머지는 실제 구현으로 통합 테스트한다.
- **Exceptional Flow**
  - **E1 (실제 서비스 접촉)**: 시뮬레이션 모드에서 실제 외부 서비스 접촉이 감지되면 호출을 차단하고 테스트를 실패 처리한다.
