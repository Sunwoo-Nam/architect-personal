# 2. 과제 개요 (Project Overview)

> Tizen TV AI Web Agent (AI Browser) 설계 문서

---

## 2.1 과제 정의

| 항목 | 내용 |
|------|------|
| **과제명** | Tizen TV AI Web Agent (AI Browser) |
| **과제 유형** | 신규 플랫폼 기능 설계 |
| **대상 플랫폼** | Samsung Tizen TV (2억+ 글로벌 설치 기반) |
| **핵심 컨셉** | TV 디바이스에서 동작하는 에이전트 하네스가 AI 모델(클라우드/온디바이스)과 연동하여, 사용자의 음성 명령을 기반으로 웹 브라우저를 자율적으로 제어하는 시스템 |

### 한 줄 정의

> 사용자의 자연어 의도를 이해하고, 웹 브라우저를 자율적으로 조작하며, TV에 최적화된 UI를 동적으로 생성하는 **On-Device 에이전트 하네스 기반의 AI Web Agent**

### 시스템 컨텍스트 — AIOS · argot · Bixby

본 시스템은 독립 제품이 아니라, Tizen OS의 AI화 이니셔티브인 **AIOS**의 구성요소로 동작한다. AIOS 안에서의 위치와 호출 흐름은 다음과 같다.

| 구성요소 | 역할 | 본 과제 범위 |
|----------|------|--------------|
| **Bixby** | 음성 입력의 진입점. 사용자 발화를 수신·전사하여 argot에 전달 | Out-of-scope (외부 음성 인식 시스템) |
| **argot** (argo-tizen) | AIOS의 **Platform Main Agent**. 사용자 질의를 해석해 적합한 Sub-Agent로 라우팅하고, 결과를 사용자에게 표시 | Out-of-scope (인터페이스 계약만 본 과제 소관) |
| **AI Web Agent (본 시스템)** | argot의 **Sub-Agent**. 질의가 브라우저 제어 태스크로 판단되면 argot이 호출 | **In-scope** |

**호출 흐름**:

1. 사용자가 Bixby를 통해 발화 → Bixby가 전사하여 argot에 전달
2. argot이 질의를 분석하여 **브라우저 제어가 필요한 태스크**로 판단하면 본 시스템(web-agent)을 호출
3. 본 시스템이 독립적으로 태스크를 수행 (에이전트 루프·브라우저 제어 등)
4. 완료 후 구조화된 결과(FR-027 스키마)를 argot에 반환
5. argot이 결과를 사용자에게 표시

**설계에 미치는 함의**:

- 본 시스템의 공개 인터페이스는 argot의 Sub-Agent 호출 계약과 호환되어야 한다 ([TC-005](04-constraints.md))
- 본 시스템의 실패는 argot으로 전파되어서는 안 된다 (FR-024 오류 격리)
- FR-001의 "외부 음성 인식 시스템"은 이 컨텍스트에서 Bixby → argot 경로를 의미한다
- argot 외에 외부 에이전트 플랫폼(MCP 클라이언트 등)에서의 호출도 지원한다 (TC-006, FR-025)
- 본 시스템은 argot 매개 호출뿐 아니라 **독립 실행**(사용자가 TV 브라우저에서 직접 사용)도 지원한다

> argot·Bixby 자체의 설계는 본 ADS의 범위가 아니다. 본 문서에서 두 시스템은 **호출자(consumer)** 로서만 다루며, 상세 인터페이스 계약은 Context View·External Interface View(08-views)에서 정의한다.

![System Context Diagram](../diagrams/05-context-diagram.drawio)

> 위 컨텍스트 다이어그램(C4 L1)은 본 시스템의 스코프 경계 — In-Scope(에이전트 하네스), out-of-scope 인접 시스템(Bixby·argot·GenUI Renderer·웹 엔진), 외부 의존(클라우드 LLM·제3자 웹 서비스·외부 에이전트 플랫폼) — 와 상호작용을 한눈에 보여준다. 스코프·인터페이스 변경 시 함께 갱신한다.

## 2.2 타겟 사용 시나리오

### 시나리오 1: 음성 기반 웹 브라우징

> "네이버에서 이번 주 날씨 알려줘"

1. 사용자가 음성으로 요청
2. 에이전트가 의도 파싱 → Headless Browser로 네이버 날씨 페이지 접근
3. 날씨 정보를 추출하여 Generative UI로 TV에 최적화된 날씨 카드 표시
4. 추가 질문 대기 ("내일은?", "부산 날씨는?")

### 시나리오 2: 자동 구매

> "쿠팡에서 항상 사던 생수 다시 주문해줘"

1. 사용자가 음성으로 구매 요청
2. 에이전트가 과거 구매 이력 또는 저장된 선호도 참조
3. Headless Browser로 쿠팡 접속 → 상품 검색 → 장바구니 추가
4. 결제 직전 단계에서 사용자에게 확인 요청 (Human-in-the-Loop)
5. 사용자 승인 후 결제 완료 → 결과 요약 표시

### 시나리오 3: 예약

> "이번 주 토요일 강남 맛집 예약해줘, 2명"

1. 사용자 요청 파싱 (날짜, 지역, 인원)
2. 에이전트가 맛집 정보 사이트 탐색 → 평점·리뷰 기반 추천 목록 생성
3. Generative UI로 추천 레스토랑 카드 표시
4. 사용자 선택 후 예약 사이트 접속 → 예약 폼 자동 입력
5. 최종 확인 후 예약 완료

### 시나리오 4: TV 브라우저 음성 제어

> "스크롤 내려줘", "저 파란 버튼 클릭해", "뒤로 가줘"

1. 현재 표시 중인 웹 페이지에 대해 직접적인 브라우저 조작 명령
2. 에이전트가 명령을 브라우저 액션으로 즉시 변환
3. 포커스 이동, 클릭, 스크롤, 뒤로가기 등 실시간 실행
4. 복합 명령도 지원 ("아래로 좀 내려서 '자세히 보기' 클릭해줘")

> **범위 조정**: 구 시나리오 5~7(Generative Web Page 기반 페이지 재가공 — 자율·에이전트 매개·사용자 명시 트리거)은 GWP 기능이 과제 범위에서 제외됨에 따라 삭제되었다. 시나리오 번호는 재배정하지 않고 1~4를 유지한다.
