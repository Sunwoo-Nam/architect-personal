# 2. 과제 개요 (Project Overview)

> Tizen TV AI Web Agent (AI Browser) 설계 문서

---

## 2.1 과제 정의

| 항목 | 내용 |
|------|------|
| **과제명** | Tizen TV AI Web Agent (AI Browser) |
| **과제 유형** | 신규 플랫폼 기능 설계 |
| **대상 플랫폼** | Samsung Tizen TV (2억+ 글로벌 설치 기반) |
| **핵심 컨셉** | TV 디바이스에서 동작하는 에이전트 하네스가 AI 모델(클라우드/온디바이스)과 연동하여, 사용자의 음성 명령을 기반으로 웹 브라우저를 자율적으로 제어하는 시스템 |

### 한 줄 정의

> 사용자의 자연어 의도를 이해하고, 웹 브라우저를 자율적으로 조작하며, TV에 최적화된 UI를 동적으로 생성하는 **On-Device 에이전트 하네스 기반의 AI Web Agent**

---

## 2.2 타겟 사용 시나리오

### 시나리오 1: 음성 기반 웹 브라우징

> "네이버에서 이번 주 날씨 알려줘"

1. 사용자가 음성으로 요청
2. 에이전트가 의도 파싱 → Headless Browser로 네이버 날씨 페이지 접근
3. 날씨 정보를 추출하여 Generative UI로 TV에 최적화된 날씨 카드 표시
4. 추가 질문 대기 ("내일은?", "부산 날씨는?")

### 시나리오 2: 자동 구매

> "쿠팡에서 항상 사던 생수 다시 주문해줘"

1. 사용자가 음성으로 구매 요청
2. 에이전트가 과거 구매 이력 또는 저장된 선호도 참조
3. Headless Browser로 쿠팡 접속 → 상품 검색 → 장바구니 추가
4. 결제 직전 단계에서 사용자에게 확인 요청 (Human-in-the-Loop)
5. 사용자 승인 후 결제 완료 → 결과 요약 표시

### 시나리오 3: 예약

> "이번 주 토요일 강남 맛집 예약해줘, 2명"

1. 사용자 요청 파싱 (날짜, 지역, 인원)
2. 에이전트가 맛집 정보 사이트 탐색 → 평점·리뷰 기반 추천 목록 생성
3. Generative UI로 추천 레스토랑 카드 표시
4. 사용자 선택 후 예약 사이트 접속 → 예약 폼 자동 입력
5. 최종 확인 후 예약 완료

### 시나리오 4: TV 브라우저 음성 제어

> "스크롤 내려줘", "저 파란 버튼 클릭해", "뒤로 가줘"

1. 현재 표시 중인 웹 페이지에 대해 직접적인 브라우저 조작 명령
2. 에이전트가 명령을 브라우저 액션으로 즉시 변환
3. 포커스 이동, 클릭, 스크롤, 뒤로가기 등 실시간 실행
4. 복합 명령도 지원 ("아래로 좀 내려서 '자세히 보기' 클릭해줘")

### 시나리오 5: Generative Web Page (GWP) 기반 페이지 재가공 — 시스템 자율 트리거

> 사용자가 직접 방문한 웹 페이지가 TV에 부적합한 경우, 시스템이 자율적으로 GWP 변환을 제안·적용

1. 사용자가 직접 URL 접근 또는 링크 이동으로 페이지 로드
2. GWP 서비스가 페이지를 사전 분석하여 레이아웃·폰트·인터랙션 요소의 TV 환경 적합성을 판정
3. 부적합 판정 시, AI가 핵심 콘텐츠를 추출하고 TV·리모컨 친화적 레이아웃(OTT 앱 스타일 D-pad 포커스 네비게이션)으로 재구성된 페이지 DOM을 산출
4. 브라우저(Chromium)가 재구성된 DOM을 렌더링하여 사용자에게 표시 (원본 보존, 사용자 토글로 즉시 원본 전환 가능)

### 시나리오 6: GWP — 에이전트 매개 트리거

> 에이전트가 태스크 결과를 사용자에게 제시할 때, 결과 페이지를 GWP 변환과 함께 표시

1. 사용자가 음성으로 에이전트에게 태스크 요청 ("쿠팡에서 생수 가격 비교해줘")
2. 에이전트가 검색·정보 수집 수행 후, 결과 페이지를 사용자에게 제시할 시점에 GWP 서비스에 위임
3. 에이전트가 사용자 의도 힌트(예: "가격 비교 중심 표시")를 GWP에 동반 전달
4. GWP가 의도 힌트를 반영해 더 정확한 레이아웃으로 재구성 (힌트가 없는 자율 트리거 대비 우선순위·강조가 정교해짐)

### 시나리오 7: GWP — 사용자 명시 요청 트리거

> 사용자가 음성 명령으로 현재 페이지의 GWP 변환을 직접 요청

1. 사용자가 일반 페이지를 보던 중 "이 페이지 TV용으로 보여줘" 발화
2. GWP 서비스가 현재 페이지에 대해 즉시 변환 수행
3. 원본/GWP 토글은 이후에도 사용자가 자유롭게 전환 가능
