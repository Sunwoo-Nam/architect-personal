## 1.1 배경 (Background)

### 1.1.1 AI Web Agent 기술 트렌드

#### 산업 전반의 Agentic AI 부상

2025년을 기점으로 AI 산업은 단순한 대화형 챗봇에서 **자율적으로 작업을 수행하는 Agent 패러다임**으로 급격히 전환되고 있다. 글로벌 AI Agent 시장 규모는 2025년 $7.84B에서 2030년까지 $52.62B에 도달할 것으로 전망된다 (CAGR 46.3%) [^1]. IT 업계 종사자 대상 설문에서 57%가 AI Agent를 프로덕션에 배포 중이며, 대기업(1만 명 이상)에서는 67%가 프로덕션 운영, 24%가 개발 후 배포 준비 단계에 있다 [^2].

#### AI Browser Agent 경쟁 현황

AI Web Agent는 모델/API 제공 단계를 넘어 **독립 AI 브라우저 제품** 및 **기존 브라우저 내 AI 통합**으로 빠르게 확장되고 있다. 2026년 4월 기준 주요 제품 현황은 다음과 같다.

### 1.1.2 AI Web Agent가 웹 콘텐츠 시장에 미치는 영향

AI Web Agent의 등장은 단순히 브라우저 자동화 도구를 넘어, 웹 콘텐츠 소비와 비즈니스 모델 전반을 구조적으로 변화시키고 있다.

#### 사용자 측면: 웹 인터랙션 방식의 근본적 변화

**소비 행동의 전환**: 소비자의 약 60%가 구매 시 AI 도구를 활용하고 있다 [^7]. 2025년 블랙 프라이데이 기준, AI 서비스를 통해 유입된 소비자는 전통적 채널 대비 구매 완료 확률이 38% 높은 것으로 나타났다 [^8]. Microsoft에 따르면 Copilot과 상호작용한 쇼핑 여정은 30분 이내 구매 확률이 53% 더 높다 [^9]. 또한, AI 채팅과 상호작용한 쇼핑객의 구매 전환율은 12.3%로, 비상호작용(3.1%) 대비 약 4배에 달한다 [^10].

**작업 효율의 비약적 향상**: AI 에이전트의 보조를 받는 쇼핑객은 구매 완료까지 47% 더 빠르게 도달한다 [^10]. 고객 서비스 영역에서는 AI 도입 후 평균 첫 응답 시간이 55% 단축된 것으로 보고되었다 [^11]. 이러한 효율 향상은 웹에서의 사용자 경험에 대한 기대치 자체를 재정의하고 있다.

**웹 콘텐츠 발견 방식의 변화**: Adobe Analytics에 따르면, 2024년 7월부터 2025년 5월 사이 미국 내 생성형 AI 기반 웹 트래픽 유입은 13배 이상 증가했다 [^12]. 기존의 검색 엔진 순위 기반 콘텐츠 발견 모델에서, AI가 콘텐츠를 요약·합성하여 사용자에게 직접 전달하는 새로운 모델로 전환이 진행 중이다.

#### 비즈니스 측면: 웹 기반 비즈니스 모델의 재편

**이커머스 패러다임 전환**: McKinsey는 AI 에이전트 기반 커머스(Agentic Commerce)가 2030년까지 글로벌 소매 지출의 $3~5T을 재배분할 것으로 전망했다 [^13]. 2025년 블랙 프라이데이에 AI를 통해 유입된 미국 소매 사이트 트래픽은 전년 대비 805% 증가했다 [^8].

**기업 운영 효율 혁신**: Forrester TEI 분석에 따르면 AI Agent 기반 고객 서비스 플랫폼은 3년간 210% ROI와 6개월 미만의 투자 회수 기간을 달성했다 [^14]. 고객 서비스 분야에서 AI는 40~60%의 자동 해결률을 달성하고 있으며 [^15], 첫 응답 시간 55% 단축 [^11] 등의 효과를 나타내고 있다.

**웹 에코시스템 구조의 변화**: GPTBot의 크롤링 볼륨은 2024년 5월 대비 2025년 5월 기준 305% 증가하여, AI 봇의 웹 트래픽 비중이 급격히 확대되고 있다 [^16]. 주요 뉴스 퍼블리셔의 79%가 AI 학습 봇을 차단하고 있으며 [^17], 동시에 WCAG 접근성 표준을 준수하는 사이트가 유기적 트래픽 23% 증가, 키워드 노출 27% 증가를 기록하는 등 AI 시대에 맞는 웹 콘텐츠 최적화가 새로운 경쟁력이 되고 있다 [^18].

### 1.1.3 TV 브라우저 및 웹 에코시스템 현황

#### Smart TV 시장 현황

글로벌 Smart TV 앱 시장은 2025년 기준 약 $50B 규모이며, 2033년까지 CAGR 15%로 성장이 전망된다 [^19]. Smart TV 플랫폼 시장은 2024년 $449.74B에서 2033년 $966.87B으로 확대될 것으로 예상된다 (CAGR 10.04%) [^20].

OS별 점유율을 보면, Android/Google TV가 글로벌 출하량 기준 약 24% 이상으로 1위를 차지하고 있으며, Samsung Tizen은 약 17~20%의 점유율을 유지하고 있다 [^20]. 미국 시장에서는 Roku OS(28%)와 Tizen(23%)이 가장 높은 사용 점유율을 보인다 [^21]. Tizen은 전 세계 약 200개국, 2억 대 이상의 Smart TV에서 구동되는 주요 플랫폼이다 [^22].

#### TV 웹 브라우저의 구조적 한계

TV 환경에서의 웹 브라우징은 다음과 같은 구조적 한계에 직면해 있다.

**입력 장치의 제약**: Smart TV 사용자의 91%가 기존 UI에 불만족하고 있으며, 가장 빈번한 불만 요인은 리모컨을 통한 포인팅 조작의 어려움이다 [^23]. 커서 키로 화면을 탐색하는 것은 마우스나 터치스크린에 비해 현저히 불편하며, 특히 텍스트 입력은 TV 플랫폼에서의 혁신을 저해하는 핵심 장벽으로 지목되고 있다.

**웹 콘텐츠 호환성**: 대부분의 웹사이트는 데스크톱/모바일 환경에 최적화되어 있어, TV의 대화면(10-feet UI)과 리모컨 기반 네비게이션에 적합하지 않다 [^24]. 반응형 웹 디자인(RWD)이 보편화되었음에도 TV 해상도와 입력 방식을 고려한 설계는 극히 드물다.

**인지 부하(Cognitive Overload)**: TV의 복잡한 네비게이션 구조는 사용자에게 인지적 과부하를 유발하며, 특히 고령 사용자층에서 포커스 조작의 어려움이 가장 빈번한 유저빌리티 문제로 보고되고 있다 [^23].

#### 웹앱 에코시스템의 중요성

TV 플랫폼에서 웹 기술은 핵심 인프라 역할을 수행한다. Tizen의 웹앱은 HTML/CSS/JavaScript 기반으로 구축되며, 플랫폼 컨테이너 내에서 웹 런타임(Web Runtime)을 통해 실행된다. 이는 개발 비용 절감(웹 표준 기반 크로스 플랫폼), 빠른 업데이트(앱스토어 배포 없이 서버사이드 업데이트), 풍부한 생태계(기존 웹 기술 자산 및 개발자 풀 활용)라는 이점을 제공한다.

그러나 TV 웹앱의 품질은 디바이스 성능, 브라우저 엔진 호환성, 리모컨 기반 UX 등 복합적 요인에 의해 모바일/데스크톱 대비 현저히 낮은 수준에 머물러 있다. 앞서 1.1.2에서 분석한 AI Web Agent의 웹 시장 영향력이 커지는 상황에서, TV 플랫폼이 이 흐름에서 소외될 경우 사용자 경험 격차가 더욱 확대될 위험이 있다.

## 1.2 과제의 필요성 (Why This Project Is Essential)

### 1.2.1 TV 웹 경험의 근본적 전환 필요

TV 브라우저의 91% 사용자 불만족률 [^23]은 단순한 UI 개선으로 해결될 수 있는 문제가 아니다. 리모컨이라는 입력 장치의 근본적 제약 위에 구축된 현재의 인터랙션 모델은 **패러다임 수준의 전환**이 필요하다. AI Web Agent는 사용자의 의도를 이해하고 웹 브라우저를 자율적으로 조작함으로써, 입력 장치의 제약을 우회하는 새로운 인터랙션 패러다임을 제시한다. 다른 플랫폼에서 AI Agent가 구매 전환율을 4배(3.1% → 12.3%) 향상시키고 [^10], 작업 완료 시간을 47% 단축한 사례 [^10]는 TV에서의 유사한 효과를 기대할 수 있는 근거가 된다.

### 1.2.2 경쟁사 대비 차별화 시급

Android/Google TV(글로벌 출하량 1위) [^20]가 Google의 Gemini 기반 AI 역량을 빠르게 통합하고 있는 상황에서, Tizen이 AI 기반 웹 경험에서 뒤처질 경우 플랫폼 경쟁력의 추가 하락이 불가피하다. 경쟁사가 제공하지 못하는 차별화된 AI 웹 경험이 필수적이다.

### 1.2.3 웹 콘텐츠 시장 변화에 대한 대응

AI Web Agent가 이커머스($3~5T 재배분 전망 [^13]), 콘텐츠 발견(AI 기반 트래픽 13배 증가 [^12]), 웹 접근성(WCAG 준수 사이트 트래픽 23% 증가 [^18]) 등 웹 생태계 전반을 재편하고 있는 상황에서, TV 플랫폼이 이 변화에 대응하지 않을 경우 다음의 위험에 직면한다.

- **사용자 기대치 격차 확대**: 모바일/데스크톱에서 AI Agent 기반의 빠르고 지능적인 웹 경험에 익숙해진 사용자가 TV 브라우저의 구식 인터랙션을 더욱 불편하게 체감
- **T-Commerce 기회 상실**: AI 기반 커머스가 주류가 되는 시점에서 TV T-Commerce의 경쟁력 약화
- **웹 콘텐츠 접근성 저하**: AI 최적화된 웹 콘텐츠가 증가하면서, AI Agent 없이는 이러한 콘텐츠를 TV에서 효과적으로 소비하기 어려워짐

### 1.2.4 시장 기회와 타이밍

AI Agent 시장이 연평균 46.3%로 급성장하고 있는 현 시점은 [^1], TV 플랫폼에 AI Web Agent를 도입하기 위한 최적의 타이밍이다. Cisco 조사에 따르면 기업의 85%가 AI Agent 파일럿을 진행 중인 상황에서 [^25], TV 분야는 아직 본격적인 AI Agent 적용이 이루어지지 않은 **블루오션 영역**이다.

CES 2026에서 Displace가 최초의 AI-Native TV를 발표하는 등 [^26] TV에서의 AI 활용이 본격화되고 있으며, 이러한 시장 흐름에 선제적으로 대응하는 것이 전략적으로 중요하다.

## 1.3 목적 (Purpose)

본 과제의 목적은 Tizen TV 플랫폼에서 **On-Device 하네스 기반의 AI Web Agent(AI Browser)**를 설계하는 것이다. 여기서 "On-Device"란 AI 모델 자체가 아닌, 웹 브라우저 제어, 콘텐츠 처리, UI 생성 등 **에이전트 실행 하네스(Agent Harness)**가 TV 디바이스에서 동작함을 의미한다. AI 모델은 클라우드 또는 온디바이스 중 상황에 따라 유연하게 선택할 수 있는 구조를 지향한다.

### 1.3.1 핵심 목적

**TV 웹 인터랙션 패러다임의 혁신**: 리모컨 기반의 수동적 웹 브라우징에서, 음성 및 자연어를 통한 AI 에이전트 기반의 자율적 웹 인터랙션으로의 전환

### 1.3.2 세부 목적

1. **음성 기반 웹 인터랙션 체계 구축**: 사용자가 음성 명령으로 웹 브라우징, 검색, 폼 입력, 구매/예약 등의 작업을 AI Agent에게 위임할 수 있는 시스템 설계

2. **TV 웹 콘텐츠 분석 및 가공 파이프라인 설계**: Headless Browser를 통해 웹 콘텐츠를 사전 로딩·분석하고, 사용자 의도에 맞는 핵심 콘텐츠를 추출·구조화하는 파이프라인 설계. 가공된 콘텐츠는 Generative UI Engine에 전달하여 TV 최적화 UI로 렌더링 (Generative UI Engine 자체는 scope 외)

4. **유연한 AI 모델 연동 아키텍처 설계**: 클라우드 LLM과 온디바이스 SLM을 상황에 따라 선택적으로 활용할 수 있는 모델 추상화 계층(Model Abstraction Layer) 설계. 에이전트 하네스는 모델 위치에 무관하게 동일한 인터페이스로 동작

5. **자동화 시나리오 지원**: 웹 기반 자동 구매, 예약, 정보 수집 등 사용자의 반복적 웹 작업을 자율적으로 수행하는 에이전트 워크플로우 설계

### 1.3.3 기대 효과

> **참고**: "현재" 수치는 출처가 확인된 경우 각주로 표기하고, 출처 미확인 시 *(추정)* 으로 명시한다. "목표" 수치는 타 플랫폼 사례를 TV 환경에 적용한 **추정치**이며, 산출 로직을 근거 열에 투명하게 기술한다.

| 지표 | 현재 (TV 브라우저) | 목표 (AI Web Agent) | 근거 |
|------|-------------------|-------------------|------|
| 웹 작업 완료 시간 | 리모컨 조작 기준 60~120초 *(추정)* | 음성 명령 기준 32~64초 | **현재**: TV 쇼핑 태스크(상품 검색→상세→장바구니) 기준 내부 추정, 공개 벤치마크 부재. **목표**: 타 플랫폼 AI 보조 시 작업 완료 47% 단축 사례 [^10]를 현재 추정치에 적용 (60초×0.53≈32초, 120초×0.53≈64초) |
| 텍스트 입력 속도 (영어 기준 분당 입력 단어 수) | 리모컨 가상키보드 ~5 단어 *(추정)* | 음성 입력 ~150 단어 | **현재**: 리모컨 D-pad 가상키보드 입력 속도에 대한 공개 연구 부재, 일반적 관찰 기반 추정. **목표**: 영어 일상 대화 발화 속도 약 150 WPM [^27] 기준, 음성 인식 정확도에 따라 실효 속도는 하락 가능 |
| 사용자 만족도 | 9% (91% 불만족 [^23]) | 개선 목표 (프로젝트 KPI로 별도 설정) | **현재**: TV UI 사용성 연구에서 91%가 불만족 응답 [^23]. **목표**: 핵심 불만 요인(리모컨 조작)이 음성 기반으로 대체될 경우 유의미한 개선을 기대하나, 정량 목표는 사용자 테스트 후 설정 |
| 구매 전환율 향상 | TV 커머스 기본 전환율 | 3~4배 향상 목표 | **현재**: TV T-Commerce 전환율에 대한 공개 데이터 제한적. **목표**: 타 플랫폼 AI 채팅 상호작용 시 전환율 4배 증가 사례 (3.1%→12.3%) [^10] 참고. TV 환경 특성상 실제 효과는 차이 가능 |

## 1.4 과제 범위 (Scope)

### In-Scope

- AI Web Agent 코어 시스템 아키텍처 설계 (On-Device 하네스)
- 모델 추상화 계층 설계 (클라우드 LLM / 온디바이스 SLM 유연 연동)
- 음성 인터페이스와 웹 브라우저 간 통합 설계
- Headless Browser 기반 웹 콘텐츠 분석 및 가공 파이프라인 설계
- 주요 사용 시나리오별 에이전트 워크플로우 설계
- TV 디바이스 자원 제약 하의 하네스 성능 최적화 전략

### Out-of-Scope

- Generative UI 렌더링 엔진 설계 및 구현
- 특정 AI 모델 학습 및 파인튜닝 방법론
- 클라우드 AI 서비스 인프라 운영 세부 사항
- 특정 웹사이트별 자동화 스크립트 구현
- DRM 콘텐츠 처리
